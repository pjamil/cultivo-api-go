# Gemini Project Overview: cultivo-api-go

This document provides a comprehensive overview of the `cultivo-api-go` project, its architecture, and key commands for development, testing, and deployment.

## 1. Project Description

`cultivo-api-go` is a RESTful API for managing cultivation-related data. It handles entities such as plants (`planta`), environments (`ambiente`), genetics (`genetica`), and users (`usuario`).

The API follows a standard layered architecture:
- **Handlers (Controllers):** Process HTTP requests.
- **Services:** Contain the core business logic.
- **Repositories:** Handle database interactions.
- **Domain (Models):** Define the data structures.

## 2. Core Technologies

- **Language:** Go `1.24.2`
- **Web Framework:** Gin
- **Database ORM:** GORM
- **Database Driver:** PostgreSQL
- **API Documentation:** Swagger (via `swaggo`)

## 3. Development Environment

The local development environment is managed via Docker Compose.

- **Database:** PostgreSQL 15
- **API Server:** The Go application runs on port `8080`.
- **Database Admin:** PGAdmin is available on port `5050`.

**Key Commands (via Makefile):**

- **Start development environment:**
  ```bash
  docker-compose up -d
  ```

- **Stop development environment:**
  ```bash
  docker-compose down
  ```

- **View logs:**
  ```bash
  docker-compose logs -f app
  ```

## 4. Build & Test

- **Build the application binary:**
  Compiles the project and creates a binary at `bin/cultivo-api`.
  ```bash
  make build
  ```

- **Run all tests:**
  ```bash
  make test
  ```

- **Build Docker image:**
  Builds a production-ready Docker image with the tag `cultivo-api`.
  ```bash
  make docker-build
  ```

## 5. API Documentation

The API documentation is auto-generated by Swagger. Once the application is running, you can access it at:

- **Swagger UI:** [http://localhost:8080/swagger/index.html](http://localhost:8080/swagger/index.html)
